## ✏️  브랜치 전략

> Git Branch 전략

- ` master `
  - 최종 브랜치
  - 기능 구현을 완전히 하였을 경우 merge
- ` dev `
  * new content 업로드 브랜치. pull request를 생성 후 통과된 폴더 및 파일만 'master'에 merge
 - ` 기능명 `
   * 새로운 기능을 추가할 때 생성하는 브랜치. 기능이 완성되면 'dev' 브랜치에 merge
